## Step 1 — back out the as-built per-position acceptance

From byteshark's accept histogram `[0, 5761, 5061, 1765, 854, 355, 214, 126, 200]`
(N=14336; bin index = tokens emitted; bin 0 never fires because every step emits ≥1
guaranteed token), the survival `S(emit≥k)` gives the per-draft-position conditional
continuation `c(j) = S(j+1)/S(j)`:

| draft position k | as-built ĉ(k) | deployed linear q(k) | full-tree model cov(k) | implied rescue ρ̂(k) |
|--:|--:|--:|--:|--:|
| 1 | **0.598** | 0.729 | 0.906 | **−0.481** |
| 2 | **0.410** | 0.759 | 0.916 | −1.449 |
| 3 | 0.498 | 0.792 | 0.928 | −1.421 |
| 4 | 0.512 | 0.822 | 0.938 | −1.739 |
| 5 | 0.603 | 0.834 | 0.943 | −1.393 |
| 6 | 0.604 | 0.835 | 0.943 | −1.406 |
| 7 | 0.613 | 0.847 | 0.947 | −1.530 |

Two structural impossibilities:

1. **The as-built sits below the deployed linear q at EVERY position.** The implied
   rank-2+ rescue `ρ̂ = (ĉ − q)/(1 − q)` is **negative at every depth** (−0.48 at
   depth 1, worse below). A tree's rescue is non-negative by construction (branches can
   only *add* acceptance), so a negative implied rescue means the build is not
   exercising the ρ-ladder at all — it is corrupting the **spine itself** below its own
   rank-1 floor. wirbel #79's measured ρ-ladder (0.4165 / 0.2655 / 0.1908) is **not
   contradicted; it is un-exercised.**

2. **Depth-1 is the cleanest diagnostic.** The depth-1 spine token is generated by the
   identical drafter forward as the linear chain — there is no tree-attention dependence
   at depth 1, and its verify is the target argmax at position 1. So in a correct build
   `ĉ(1)` **must equal** `q[1] = 0.7287` (= `pvecs_depth1[1]`). The as-built **0.598** is
   a 13.1pp deficit: 40.2% of steps accept zero draft tokens vs 27.1% for the bare linear
   chain and 9.4% for the full tree. A verify/dispatch defect, full stop — it cannot be
   draft quality (depth-1 draft is identical to linear) and cannot be a ceiling.

## Step 2 — classify the defect

Reference ladder (all tokens-emitted-per-step, bonus included):

```
  model full tree    5.207   <- analytical, de-risked (#79/#85/#91/#92)
  model spine-only   4.177   <- rank-1 depth-9 spine
  deployed linear    3.844   <- MEASURED on the real stack (hard floor)
  as-built tree      2.102   <- byteshark, this build
```

Gap = 5.207 − 2.102 = **3.105 tok**, decomposed:

| segment | tokens | % of gap | attribution |
|---|--:|--:|---|
| **sub-linear collapse** (3.844 → 2.102) | **1.742** | **56.1%** | **(A) build defect — PROVABLE** |
| spine depth-9 ext (4.177 → 3.844) | 0.333 | 10.7% | de-risked premium (unrealized) |
| branch premium (5.207 → 4.177) | 1.030 | 33.2% | de-risked premium (unrealized) |

- **(A) truncated branch exploration — DOMINANT.** The salvage counter fires at
  **0.358/non-full-step**, *above* the model-expected 0.336 (ratio 1.07): the drafter
  **is** producing correct rank-2+ candidates and the walk **is** dispatching them. Yet
  accept-length collapses below the spine. The only consistent mechanism: the salvage
  walk recovers the immediate rescue node but **does not descend its sub-path** — every
  salvage terminates early. Confirmed by the full-tree reach rate: **1.10%** of steps
  reach full depth vs the model's ~60.8% — the walk almost never traverses past the
  shallow front.
- **(B) draft-quality collapse on the tree layout — BOUNDED SMALL.** Salvage firing at
  the model rate bounds (B) small at the front (depths 1–2): if star-attn numerics had
  collapsed draft quality, the rank-2+ candidates would be *wrong* and salvage would be
  *low*, not at 1.07× model. A residual deep-tail (B) cannot be excluded from these
  aggregate counters (few counts past depth 5) — this is the one piece the re-measure
  must close.
- **(C) eager-mode artifact — ~0% of the accept-length gap.** `attn_py_calls/step=37`
  is enforce-eager python dispatch: it changes per-step **overhead** (the TPS axis,
  denken #97/#85), not the verify numerics/argmax, so it does not cap accept-length.
  Diagnostic-only, resolves under the graph path; not part of this gap.
- **(D) optimistic model — 0%.** fern #92's independence result (realized 5.208 =
  independent 5.207, **+0.025%**, within ±3% across three cross-checks) already confirmed
  the 5.207 ceiling is sound for the **ideal** tree under real correlated draws. The
  as-built gap is a *realization* failure, not a *model* failure.

**`tree_accept_length_gap_explained_pct = 100%`** attributable to fixable build/dispatch
defects (A dominant; B bounded-small; C≈0 on this axis); **0% is the (D) ceiling.** The
unassailable lower bound is **56.1%** — the part below the measured linear floor, which
no acceptance-model story can produce.

## Caveats

1. **Two distinct blockers, do not conflate.** This report diagnoses the **accept-length**
   collapse (build/walk defect, the 2.10 number). wirbel #93's **fp32 star-attn**
   requirement is a separate **greedy-validity** blocker (relerr-1e-3 flips 0.59% of
   tokens). depth-1 doesn't use star-attn, and 0.59% flips can't produce a 13pp depth-1
   deficit, so the two are independent. Both must be fixed before quota.
2. **Counters are byteshark's, imported as-is.** The histogram snapshot is at step 14336
   (terminal counters at 14848); the small 14336-vs-14848 offset does not move the
   aggregate (2.10 « 3.844 is robust to any single-bin or snapshot quirk).
3. **Residual deep-tail (B) unbounded by these counters.** The front (depths 1–2, where
   most mass and the smoking gun live) is pinned tightly; depths 5–8 have few counts, so a
   residual star-attn draft-quality loss there is possible — that is exactly what the
   per-position branch-hit re-measure must close. The GREEN gate is on the *diagnosis*
   (defect, not ceiling); the *realized post-fix number* awaits the re-measure.
4. **CPU-only, no GPU/HF.** Pure arithmetic on committed counters + the validated
   acceptance machinery (`treeshape_measured_accept`); E[T] anchors reproduce wirbel's
   5.207 to 4.6e-5.
